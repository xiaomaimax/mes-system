## 🚀 快速开始流程

### 方案A: 一键自动部署（推荐新手）

```bash
# 1. 下载部署脚本
wget https://your-repo/deploy.sh
chmod +x deploy.sh

# 2. 运行脚本
./deploy.sh

# 3. 按照提示输入信息
# - MySQL root密码
# - MES数据库用户密码
# - 服务器IP或域名

# 4. 等待部署完成
# 预计时间：15-20分钟
```

**优点**:
- 快速简单
- 自动化程度高
- 适合新手

**缺点**:
- 无法自定义配置
- 遇到问题难以调试

---

### 方案B: 快速手动部署（推荐有经验的人）

```bash
# 1. 阅读快速开始指南
cat DEPLOYMENT_QUICK_START.md

# 2. 按照指南逐步执行命令
# 预计时间：20-30分钟

# 3. 使用检查清单验证部署
cat DEPLOYMENT_CHECKLIST.md
```

**优点**:
- 可以自定义配置
- 便于调试和问题排查
- 学习系统架构

**缺点**:
- 需要手动执行多个步骤
- 需要一定的Linux知识

---

### 方案C: 详细学习部署（推荐深入学习）

**优点**:

- 便于后续维护

**缺点**:

## 📋 文档使用建议

### 首次部署

1. **第一步**: 阅读本文档（5分钟）
2. **第二步**: 选择部署方案
   - 如果是新手：使用方案A（一键部署）
   - 如果有经验：使用方案B（快速部署）
   - 如果想深入学习：使用方案C（详细学习）
3. **第三步**: 按照选定方案部署
4. **第四步**: 使用检查清单验证部署
5. **第五步**: 阅读维护指南

### 部署遇到问题

1. **第一步**: 查看维护指南中的"常见问题"部分
2. **第二步**: 查看完整部署指南中的"故障排查"部分
3. **第三步**: 查看应用日志：`pm2 logs mes-api`
4. **第四步**: 查看系统日志：`sudo tail -f /var/log/syslog`
5. **第五步**: 如果仍无法解决，提交问题到GitHub

### 日常维护

1. **每日**: 运行daily-check.sh脚本
2. **每周**: 阅读维护指南中的"每周维护任务"部分
3. **每月**: 阅读维护指南中的"每月维护任务"部分
4. **遇到问题**: 查看维护指南中的"常见问题"部分

## 🎯 部署目标

部署完成后，应该达到以下目标：

### 功能目标
- ✅ 前端应用可以访问
- ✅ 后端API可以正常工作
- ✅ 数据库可以正常连接
- ✅ 所有模块功能正常

### 性能目标
- ✅ API响应时间 < 500ms
- ✅ 前端加载时间 < 2s
- ✅ 数据库查询时间 < 100ms
- ✅ 系统内存占用 < 1GB

### 可靠性目标
- ✅ 应用自动重启
- ✅ 数据库自动备份
- ✅ 系统监控告警
- ✅ 日志自动轮转

### 安全目标
- ✅ 防火墙已配置
- ✅ 文件权限正确
- ✅ 密钥已保护
- ✅ SSL证书已配置（如使用HTTPS）
