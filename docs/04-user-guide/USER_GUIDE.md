## 系统概述

### 系统简介
MES制造执行系统是一套完整的制造执行管理解决方案，涵盖生产、质量、设备、库存等全流程管理。

### 主要特点
- **全流程管理**: 从生产计划到质量控制的完整流程
- **实时监控**: 生产状态和设备运行实时监控
- **数据驱动**: 基于数据的决策支持和分析
- **权限控制**: 细粒度的用户权限管理
- **集成能力**: 强大的第三方系统集成

### 系统架构
```
用户界面层 (React + Ant Design)
    ↓
业务逻辑层 (Node.js + Express)
    ↓
数据存储层 (MySQL + Redis)
    ↓
设备接口层 (OPC/Modbus/TCP)
```

## 核心功能模块

### 1. 生产管理模块

#### 功能概述
管理生产计划、工单执行、生产监控等核心生产业务。

#### 主要功能
- **车间计划管理**: 制定和调整生产计划
- **生产任务管理**: 工单创建、分配、执行
- **生产执行管理**: 实时生产状态监控
- **工作报告管理**: 生产数据记录和汇总
- **日报管理**: 生产日报生成和分析
- **设备责任管理**: 设备操作员分配
- **班次排程管理**: 生产班次安排
- **产线物料管理**: 生产线物料配送
- **生产主数据管理**: 产品、工艺等基础数据

#### 操作流程
```
1. 创建生产计划
   ↓
2. 生成生产工单
   ↓
3. 分配生产任务
   ↓
4. 执行生产作业
   ↓
5. 记录生产数据
   ↓
6. 生成生产报告
```

### 2. 质量管理模块

#### 功能概述
全流程质量控制，包括来料检验、过程检验、成品检验等。

#### 主要功能
- **IQC检验**: 来料质量检验
- **PQC检验**: 过程质量检验
- **FQC检验**: 成品质量检验
- **OQC检验**: 出货质量检验
- **不良原因管理**: 质量问题分析
- **检验标准管理**: 质量标准维护
- **不良记录管理**: 质量问题记录
- **批次追溯**: 产品质量追溯

#### 质量控制流程
```
原料入库 → IQC检验 → 合格入库
    ↓
生产过程 → PQC检验 → 过程控制
    ↓
成品检验 → FQC检验 → 合格入库
    ↓
出货检验 → OQC检验 → 合格出货
```

### 3. 设备管理模块

#### 功能概述
设备全生命周期管理，包括档案、维护、故障等管理。

#### 主要功能
- **设备档案**: 设备基础信息管理
- **设备维护**: 预防性维护计划和执行
- **设备检查**: 日常点检和定期检查
- **设备维修**: 故障维修记录和管理
- **设备关系**: 设备层级和关联关系
- **设备主数据**: 设备分类和参数

#### 设备管理流程
```
设备采购 → 设备验收 → 建立档案
    ↓
投入使用 → 日常点检 → 预防维护
    ↓
故障发生 → 故障维修 → 维修记录
    ↓
设备报废 → 资产处置 → 档案归档
```

### 4. 库存管理模块

#### 功能概述
物料全流程管理，包括采购、入库、出库、盘点等。

#### 主要功能
- **库存主数据**: 物料基础信息管理
- **出入库管理**: 物料出入库操作
- **库存调拨**: 库存转移和调配
- **库存盘点**: 定期库存盘点
- **库存报表**: 库存分析和报告
- **库存设置**: 库存参数配置
- **备件流转**: 备件专项管理
- **外协备件**: 外协物料管理
- **备件预警**: 库存预警提醒

## 常用操作流程

### 生产作业流程

#### 1. 生产计划制定
```
步骤1: 进入"生产管理" → "车间计划管理"
步骤2: 点击"新增计划"按钮
步骤3: 填写计划信息：
   - 产品信息
   - 计划数量
   - 计划时间
   - 生产线分配
步骤4: 保存并提交审核
步骤5: 审核通过后发布执行
```

#### 2. 工单执行
```
步骤1: 进入"生产管理" → "生产任务管理"
步骤2: 选择待执行工单
步骤3: 点击"开始生产"
步骤4: 实时录入生产数据：
   - 产量数据
   - 质量数据
   - 设备状态
   - 异常情况
步骤5: 完成生产后点击"完工"
```

### 质量检验流程

#### 1. 来料检验(IQC)
```
步骤1: 进入"质量管理" → "IQC检验"
步骤2: 扫描或输入物料批次号
步骤3: 选择检验标准
步骤4: 执行检验项目：
   - 外观检验
   - 尺寸检验
   - 性能检验
步骤5: 录入检验结果
步骤6: 判定合格/不合格
步骤7: 生成检验报告
```

#### 2. 过程检验(PQC)
```
步骤1: 进入"质量管理" → "PQC检验"
步骤2: 选择生产工单
步骤3: 按检验频次抽样
步骤4: 执行检验项目
步骤5: 记录检验数据
步骤6: 异常情况处理
步骤7: 更新工单质量状态
```

### 设备维护流程

#### 1. 预防性维护
```
步骤1: 进入"设备管理" → "设备维护"
步骤2: 查看维护计划
步骤3: 选择到期维护项目
步骤4: 执行维护作业：
   - 清洁保养
   - 润滑加油
   - 紧固检查
   - 功能测试
步骤5: 记录维护内容
步骤6: 更新设备状态
步骤7: 安排下次维护
```

#### 2. 故障维修
```
步骤1: 设备故障报告
步骤2: 进入"设备管理" → "设备维修"
步骤3: 创建维修工单
步骤4: 故障诊断分析
步骤5: 制定维修方案
步骤6: 执行维修作业
步骤7: 维修验证测试
步骤8: 记录维修报告
步骤9: 设备恢复生产
```

## 权限和安全

### 用户权限体系

#### 权限层级
```
系统级权限 (超级管理员)
    ↓
部门级权限 (部门管理员)
    ↓
功能级权限 (普通用户)
    ↓
数据级权限 (操作员)
```

#### 角色定义
- **超级管理员**: 系统全部功能权限
- **部门管理员**: 本部门业务管理权限
- **技术管理员**: 技术相关功能权限
- **普通用户**: 基本业务操作权限
- **只读用户**: 数据查看权限

#### 部门权限配置
不同部门用户登录后看到不同的功能模块：

- **生产部**: 生产管理、设备管理、库存管理、报表分析
- **质量部**: 质量管理、工艺管理、报表分析
- **技术部**: 工艺管理、设备管理、系统集成、报表分析
- **信息部**: 全部模块访问权限

### 安全设置

#### 密码策略
- 密码长度：8-20位
- 密码复杂度：包含大小写字母、数字、特殊字符
- 密码有效期：90天
- 历史密码：不能重复使用最近5次密码

#### 登录安全
- 登录失败锁定：连续5次失败锁定30分钟
- 会话超时：无操作30分钟自动退出
- 单点登录：同一账号只能在一处登录
- IP限制：可配置允许登录的IP地址段

#### 操作审计
- 登录日志：记录所有登录行为
- 操作日志：记录关键业务操作
- 数据变更：记录重要数据修改
- 系统日志：记录系统运行状态

## 数据管理

### 数据导入

#### 支持格式
- Excel文件 (.xlsx, .xls)
- CSV文件 (.csv)
- 文本文件 (.txt)

#### 导入流程
```
步骤1: 下载导入模板
步骤2: 按模板格式准备数据
步骤3: 选择导入功能
步骤4: 上传数据文件
步骤5: 数据格式验证
步骤6: 预览导入数据
步骤7: 确认执行导入
步骤8: 查看导入结果
```

#### 注意事项
- 数据格式必须严格按照模板
- 必填字段不能为空
- 数据重复时按配置规则处理
- 导入失败数据会生成错误报告

### 数据导出

#### 导出格式
- Excel格式：适合数据分析
- PDF格式：适合打印报告
- CSV格式：适合系统对接

#### 导出范围
- 当前页面数据
- 查询结果数据
- 全部数据（需要权限）
- 自定义字段选择

### 数据备份

#### 自动备份
- 每日自动备份：凌晨2点执行
- 备份保留期：30天
- 备份存储位置：服务器备份目录

#### 手动备份
- 管理员可随时执行备份
- 支持增量备份和全量备份
- 备份文件可下载到本地

## 📞 技术支持

### 常见问题

#### 登录问题
**Q: 忘记密码怎么办？**
A: 联系系统管理员重置密码，或使用密码找回功能。

**Q: 账号被锁定怎么办？**
A: 等待30分钟自动解锁，或联系管理员手动解锁。

#### 操作问题
**Q: 数据保存失败怎么办？**
A: 检查必填字段是否完整，网络连接是否正常。

**Q: 页面显示异常怎么办？**
A: 刷新页面，清除浏览器缓存，或更换浏览器。

#### 权限问题
**Q: 看不到某个功能模块？**
A: 确认用户角色和部门权限配置是否正确。

**Q: 无法执行某个操作？**
A: 检查是否有相应的操作权限。
