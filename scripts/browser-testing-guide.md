# MES系统浏览器功能测试指南

## 🚀 开始测试

### 第一步：确认服务器状态
1. 确认后端服务器运行在 http://localhost:5002
2. 确认前端服务器运行在 http://localhost:3000
3. 两个服务器都应该正常运行，无错误信息

### 第二步：打开浏览器开始测试
1. 打开浏览器（推荐Chrome或Edge）
2. 访问 http://localhost:3000
3. 按照以下测试流程进行

## 📋 详细测试流程

### 1. 登录功能测试 (5分钟)
```
访问地址: http://localhost:3000
预期: 自动跳转到登录页面

测试账户:
- 管理员: admin / admin123
- 经理: manager / manager123  
- 技术员: tech / tech123
- 操作员: user / user123

测试步骤:
1. 输入 admin / admin123
2. 点击登录按钮
3. 验证登录成功，跳转到首页
4. 检查右上角显示用户信息
```

### 2. 首页概览测试 (3分钟)
```
测试内容:
- 生产状态统计数据
- 设备运行状态
- 质量指标
- 库存预警信息
- 快速操作按钮

验证要点:
- 数据显示正常
- 图表渲染正确
- 按钮点击有响应
```

### 3. 导航菜单测试 (10分钟)
```
依次点击左侧菜单项，验证每个模块能正常加载:

✅ 首页 - 概览数据
✅ 工艺管理 - 工艺流程、参数设置
✅ 生产管理 - 生产任务、车间计划
✅ 设备管理 - 设备状态、维护计划
✅ 质量管理 - 质量检验、批次追溯
✅ 库存管理 - 库存状态、出入库
✅ 人员管理 - 员工信息、考勤管理
✅ 系统集成 - 数据接口、同步状态
✅ 报表中心 - 各类统计报表
✅ 系统设置 - 用户管理、权限设置
```

### 5. 权限控制测试 (10分钟)
```
测试步骤:
1. 退出当前账户
2. 使用不同角色账户登录
3. 验证权限控制

测试账户权限:
- admin: 全部功能权限
- manager: 生产、质量、设备管理权限
- tech: 生产执行、设备维护权限  
- user: 基础查看权限

验证要点:
- 菜单显示符合权限
- 功能访问受限正确
- 敏感操作需要权限
```

### 6. 响应式设计测试 (5分钟)
```
测试步骤:
1. 调整浏览器窗口大小

3. 验证页面适配效果

验证要点:
- 页面布局自适应
- 移动端操作友好
- 内容显示完整
```

## 🔍 测试重点关注

### 关键功能点
1. **用户认证** - 登录/登出功能
2. **数据显示** - 各模块数据正确显示
3. **导航切换** - 菜单导航流畅
4. **权限控制** - 角色权限正确
5. **响应速度** - 页面加载快速

### 常见问题检查
1. **页面空白** - 检查控制台错误
2. **数据不显示** - 检查网络请求
3. **功能无响应** - 检查JavaScript错误
4. **样式异常** - 检查CSS加载
5. **权限错误** - 检查用户角色

## 📊 测试结果记录

### 成功标准
- 所有核心功能正常运行
- 用户体验流畅
- 无严重错误或崩溃
- 权限控制正确
- 响应速度满足要求

### 问题分类
- **严重**: 功能无法使用，系统崩溃
- **一般**: 功能异常，影响使用
- **轻微**: 界面问题，不影响功能
- **建议**: 用户体验优化建议
